# Elicitation Methods

## Available Methods

### 1. Structured Interview
- Best for: Requirements gathering, stakeholder input
- Format: Predefined questions with follow-ups
- Output: Documented responses with priorities

### 2. User Story Mapping
- Best for: Feature discovery, workflow definition
- Format: Visual mapping of user journeys
- Output: Prioritized user stories

### 3. MoSCoW Prioritization
- Best for: Feature prioritization, scope definition
- Format: Must/Should/Could/Won't categories
- Output: Prioritized feature list

### 4. Rapid Prototyping
- Best for: UI/UX validation, concept testing
- Format: Quick mockups or code prototypes
- Output: Validated design decisions

### 5. 5 Whys Analysis
- Best for: Root cause analysis, problem definition
- Format: Iterative "why" questioning
- Output: Root cause identification

### 6. SWOT Analysis
- Best for: Strategic decisions, competitive analysis
- Format: Strengths/Weaknesses/Opportunities/Threats
- Output: Strategic positioning insights
